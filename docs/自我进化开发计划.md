# 自我进化开发计划

依据 [自我进化规则](./自我进化规则.md) 拆分为可交付阶段。本文档为实施路线图与当前进度。

## 目标

- **CLI 为本**：生产对话始终可用已发布二进制；进化失败可回退。
- **源码可拓**：可选挂载 Git 源码中心，自动拉取、编译、验证候选 CLI。
- **进化有预算**：复用 `AssistantGlobalSettings` 中 `evolution_*` 字段记账。
- **外环在 Git**：Issue/PR 协作在 E2～E3 落地；E0～E1 先打通本地与 API。

## 阶段与状态

| 阶段 | 内容 | 状态 |
|------|------|------|
| **E0** | 配置 schema、`data/evolution/` 目录、`registry.json`、运行日志、HTTP 读写配置 | ✅ 本迭代 |
| **E1** | 自动步骤 1～2：拉源码、备份 CLI、编译、健康检查、回退 | ✅ 本迭代 |
| **E2** | 步骤 3～4：源码分析、查/提 Issue（LLM + 注册表） | ✅ 本迭代 |

## 目录布局（E0）

```
{SEVEN_CHAT_AGENT_DATA}/evolution/
  settings.json          # runtime / source / evolution 配置
  registry.json          # 本地 issue 注册表
  cli-backup/            # 基础 CLI 时间戳备份
  workspaces/            # 各源码中心 git 工作副本
  runs/                  # 每次任务 JSON 日志
```

## 配置模型（E0）

见 `seven-chat-agent-core::evolution::EvolutionSettings`：

- `runtime_mode`: `cli` | `source`
- `cli.binary_path` / `cli.preset`
- `source`: 远程 URL、分支、build/test 命令、产物路径
- `evolution`: enabled、max_concurrent_tasks、审批开关、trusted_orgs

进化 token 池继续使用助理全局设置中的 `evolution_token_budget_*` / `evolution_tokens_used`。

## API（E0～E2）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/evolution/settings` | 读取配置 |
| PUT | `/evolution/settings` | 保存配置 |
| GET | `/evolution/registry` | 本地 issue 注册表 |
| GET | `/evolution/runs` | 运行历史列表 |
| GET | `/evolution/runs/:id` | 单次运行详情 |
| POST | `/evolution/runs/sync-source` | 步骤 1：git 同步 |
| POST | `/evolution/runs/build-cli` | 步骤 2：备份+编译+健康检查 |
| POST | `/evolution/runs/pipeline-sync-build` | 1+2 串联 |
| POST | `/evolution/runs/analyze-source` | 步骤 3：静态扫描 + 可选 LLM 增强 |
| POST | `/evolution/runs/sync-issues` | 步骤 4：GitHub 查重/创建（可传 `report`） |
| POST | `/evolution/runs/pipeline-analyze-issues` | 3+4 串联 |

## E1 流程说明

1. **sync-source**：在 `workspaces/<id>/` 执行 clone 或 fetch+checkout；记录 commit。
2. **build-cli**：
   - 将 `cli.binary_path` 复制到 `cli-backup/<timestamp>/`
   - 在 worktree 执行 `source.build`
   - 对 `built_binary_path` 执行 `--help` smoke
   - 成功则写入 run 日志；失败不替换 active，保留 backup 路径供回退

## E2 说明（本迭代）

- **analyze-source**：遍历工作区（跳过 `target`/`node_modules` 等），产出 `OptimizationReport`（TODO/unwrap/大文件等）；`evolution.analyze_use_llm` 为 true 时用 Judge LLM 补充条目。
- **sync-issues**：按 `issue_similarity_threshold` 与本地注册表、GitHub search 查重；默认 `require_approval_before_create_issue` 为 true 时仅登记待审批。
- Token：`GITHUB_TOKEN` 或 `evolution.github_token`；进化预算走助理全局 `evolution_token_budget_*`。
- 运行产物写入 `runs/<id>.artifact.json`（`EvolutionRunLog.artifact`）。

## E3 预告
